Chapter 10 : Connection Management

Working with databases in any application involves these steps:
	1.	Create a Connection
	2.	Perform DB Operations
	3.	Close / Destroy the Connection

* Creating and destroying database connections repeatedly
  is very expensive, especially in high-load applications / enterprise apps
* To improve performance, we use Connection Pooling.

* Connection Pooling will reuse database connections instead of creating new ones every time.

* Hibernate provides 3 ways to implement connection pooling:

2. Third-Party Connection Management Libs(Static)

Hibernate supports several external connection pool libraries.
One of the most widely used is C3P0.

3. Application Server Provided Pooling (Dynamic)

(Like JBoss, WebLogic, WebSphere, Tomcat )

* Here the server maintains the pool, and Hibernate will fetch connections.
* Most efficient for large enterprise apps.
